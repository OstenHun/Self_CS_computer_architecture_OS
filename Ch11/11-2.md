## CPU 스케줄링 알고리즘 종류

### 선입 선처리 스케줄링 (FCFS 스케줄링 ; First Come First Served Scheduling)

→ 준비된 큐에 삽입된 순서대로 프로세스들을 처리하는 비선점형 스케줄링 방식; CPU 먼저 요청한 프로세스부터 CPU 할당

단점 : 호위 효과 (convoy effect) → CPU를 오래 사용하는 프로세스가 먼저 도착하면 다른 프로세스는 계속 기다려야함.

### 최단 작업 우선 스케줄링 (SJF 스케줄링 ; Shortest Job First Scheduling)

→  준비 큐에 삽입된 프로세스들 중 CPU 이용 시간의 길이가 가장 짧은 프로세스부터 실행하는 방식

선점형, 비선점형 모두 구현 가능

### 라운드 로빈 스케줄링 (rr 스케줄링 ; round robing scheduling)

→ 정해진 타임 슬라이스만크의 시간 동안 돌아가며 CPU를 이용하는 선점형 스케줄링.

- 타임 슬라이스 : 각 프로세스가 CPU를 사용할 수 있는 정해진 시간
    - 타임슬라이스 지나치게 큼 → 선입 선처리 스케줄링과 같아져 호위 효과 발생 가능
    - 타임슬라이스 지나치게 작음 → 문맥 교환에 발생하는 비용이 커져 CPU는 프로세스를 처리하는 일보다 프로세스 전환에 큰 자원을 사용.

### 최소 잔여 시간 우선 스케줄링 (SRT 스케줄링 ; Shortest Reamining Time)

→ SJF 알고리즘과 RR 알고리즘을 합친 스케줄링 방식

→ 프로세스들은 정해진 타임 슬라이스만큼 CPU를 사용하되, CPU를 사용할 다음 프로세스로는 남아있는 작업 시간이 가장 적은 프로세스가 선택.

### 우선순위 스케줄링

→ 프로세스들에 우선순위를 부여, 가장 높은 우선순위를 가진 프로세스부터 실행하는 알고리즘.

- 우선순위가 같은 경우 선입 선처리로 스케줄링.

단점 : **기아 (Starvation)** 현상 발생 가능. 

→ 우선순위가 높은 프로세스만 먼저 실행 되어 우선 순위가 낮은 프로세스의 실행은 계속 밀리는 현상

기아 현상 방지 기법: **에이징 (aging)** 

→ 오랫동안 대기한 프로세스 우선순위를 높이는 방식.

### 다단계 큐 스케줄링

- 우선순위 스케줄링의 발전된 형태.
- 우선순위별로 준비 큐를 여러 개 사용하는 방식.
    - 프로세스 유형별로 우선순위를 구분해 실행 가능. (CPU 집중, 입출력 집중, 백그라운드, 사용자와 상호작용 잦은 프로세스)
    - 우선순위가 가장 높은 큐에 있는 프로세스들 먼저 처리, 가장 높은 큐가 비어 있으면 그다음 우선순위 큐에 있는 프로세스 처리.

- 큐 별로 타임 슬라이스 여러 개 지정 가능
- 큐마다 다른 스케줄링 알고리즘 사용 가능.

### 다단계 피드백 큐 스케줄링

- 다단계 큐 스케줄링과 비슷 → But, 프로세스들이 큐 사이 이동 가능.
    - 새로 준비 상태가 된 프로세스가 있다면 우선순위가 가장 높은 우선순위 큐에 삽입되고 일정시간(타임 슬라이스)동안 실행.
    - 프로세스가 해당 큐에서 실행이 끝나지 않는다면 다음 우선순위 큐에 삽입 돼 실행.
    - 또 실행이 안 끝나면 또 다음 우선순위 큐에 삽입. 결국 CPU 오래 사용해야 하는 프로세스는 우선순위가 낮아진다.
        - CPU 오래 사용해야 하는 CPU 집중 프로세스들은 우선순위 낮아짐.
        - CPU 비교적 적게 사용하는 입출력 프로세스들은 자연스럽게 우선순위가 높아짐.
        - → 에이징 기법 활용하여 기아 현상 예방 가능.

⇒  어떤 프로세스의 CPU 이용 시간이 길면 낮은 우선순위 큐로 이동시키고, 어떤 프로세스가 낮은 우선순위 큐에서 너무 오래 기다리면 높은 우선순위 큐로 이동시킬 수 있는 알고리즘.

## 마무리

### 키워드

- 선입 선처리 스케줄링
    - 준비 큐에 삽입된 순서대로 CPU를 할당.
- 최단 작업 우선 스케줄링
    - 준비 큐에 삽입된 프로세스들 중 CPU 사용 시간의 길이가 가장 짧은 프로세스부터 CPU를 할당.
- 라운드 로빈 스케줄링
    - 정해진 시간만큼만 돌아가며 CPU를 할당.
- 우선순위 스케줄링
    - 가장 높은 우선순위를 가진 프로세스에 CPU를 할당.
- 다단계 피드백 큐 스케줄링
    - 프로세스들이 큐 사이를 이동할 수 있는 다단계 큐 스케줄링.
